# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`dfc_flutter` is a Flutter **package** (not an app), publishing miscellaneous widgets and utilities. It is consumed by other apps via a git dependency. There is no `main()`, no runnable target, and `publish_to: "none"` in `pubspec.yaml`.

## Common commands

All scripts live in `tools/` and are thin wrappers around Flutter / Dart commands.

- `./tools/model_build.sh` — run `build_runner` to regenerate `*.g.dart` files (json_serializable). Required after editing any model with `@JsonSerializable`.
- `./tools/model_clean.sh` — clean build_runner caches.
- `./tools/translate.sh` — runs `lib/src/translator_tool.dart` (auto-translates `app_en.arb` → other languages via an HTTP translator), then `flutter gen-l10n`. Re-run whenever `lib/src/arb/app_en.arb` changes.
- `./tools/pub.sh` / `./tools/spub.sh` — `flutter pub upgrade` (spub also bumps major versions and tightens).
- `./tools/outdated.sh` — `flutter pub outdated --no-transitive`.
- `flutter analyze` — lint using `dfc_lints` rules (configured via `analysis_options.yaml`).

There are no tests in this repo.

## Three public entry points (layered)

The package exports through three `.dart` files at `lib/`, layered so consumers pick the smallest viable surface:

1. `dfc_flutter_web_lite.dart` — pure-Dart / pure-Flutter widgets, dialogs, extensions, themes, hive, prefs, SEO. Safe for web.
2. `dfc_flutter_web.dart` — re-exports `_lite` plus `google_fonts/font_utils`. Web-safe with fonts.
3. `dfc_flutter.dart` — re-exports `_web` plus IO-only modules: `file_system/`, `requests/`, `widgets/image/`, `widgets/video_player_widget`, `utils/create_icon_screen`.

When adding a new public symbol, add the export to the **lowest layer** it can live in. Do not export web-incompatible code (`dart:io`, file system, video) from `_lite` or `_web`.

## Web vs. IO platform split

`lib/src/web_utils/` uses Dart conditional imports to keep the package web-compatible:

```
web_utils.dart
  → web_utils_stub.dart           (default)
  → web_utils_js.dart   if dart.library.js_interop
  → web_utils_io.dart   if dart.library.io
```

When adding APIs that touch `dart:html`/`dart:js_interop` or `dart:io`, follow the same stub + conditional-import pattern. The interface is `web_utils_interface.dart`; the public class `WebUtils` wraps the resolved `WebUtilsImp`.

## Localization

- Source-of-truth strings: `lib/src/arb/app_en.arb`. Other language ARBs are machine-generated by `translator_tool.dart`.
- Generated Dart lives in `lib/l10n/app_localizations*.dart` (output configured by `l10n.yaml`).
- Consumers wire the package's localizations via `dfcFlutterLocDelegates` and `dfcFlutterLocales` from `lib/l10n/app_locs.dart`.
- After editing `app_en.arb`, run `./tools/translate.sh`. Do not hand-edit non-English ARBs — they will be overwritten.

## Code generation

Models annotated with `@JsonSerializable` (under `lib/src/models/` and `lib/src/wp_api/`) have committed `*.g.dart` siblings. After editing any annotated model, run `./tools/model_build.sh`.

## Dependencies

Several dependencies are pinned to git forks owned by the same author and must resolve over the network:
- `dfc_dart` (sgehrman/dfc_dart)
- `flutter_animate` (sgehrman/flutter_animate)
- `dfc_lints` (sgehrman/dfc_lints) — provides the lint rules
- `dart_code_metrics` (JonasWanke fork) — required alongside `dfc_lints`

Hive uses the community-edition packages (`hive_ce`, `hive_ce_flutter`), not the original `hive`.
